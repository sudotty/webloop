# WebLoop —— ノーコード・ローカルファーストのブラウザ自動化

**繰り返しのブラウザ操作を一度だけ記録し、いつでも再生・スケジュール実行。**
レガシーな ERP・CRM・OA システム、社内ネットワーク、レポートポータル向けの軽量な RPA 代替 —— コード不要、クラウド不要、LLM 不要。

## 😩 課題

社内の業務ソフト —— ERP、CRM、OA、イントラネット、レポートポータル —— の多くには **API がありません**。
そのため、人が API の代わりになります。毎朝だれかが同じページを開き、昨日の日付を選び、
同じフィルターを適用し、「検索」を押して遅いテーブルの読み込みを待ち、「エクスポート」を押して
ファイルをメールで送る。来る日も来る日も。

重量級の RPA スイート（UiPath、Power Automate）は高価で、トレーニングが必要で、デスクトップ
インストールを求めます。純粋な「AI エージェント」はハルシネーションを起こし、決定論的ではありません。
「すべてのサイトのデータを読み取る」ことを要求するクラウド型レコーダーは、情報セキュリティ部門に拒否されます。

## 💡 解決策

> **WebLoop はブラウザの DOM を API として扱います。** クリックできるなら、自動化できます。

これは軽量な Chrome 拡張機能で、**あなたの実際のクリックを一度だけ記録**し、決定論的に再生します ——
オンデマンドでもスケジュールでも、すべて**あなたのマシン上で**動作します。

## ✨ 現在利用できる機能

| | 機能 | 内容 |
| :-- | :--- | :--- |
| ✅ | **ノーコード記録** | クリック、入力、セレクト、ホバー、ダブルクリック、チェックボックス、ダウンロードを捕捉。 |
| ✅ | **決定論的な再生** | 複数のロケーター（CSS、XPath、安定テキスト、aria-label）＋信頼度スコア —— ハルシネーションするエージェントに依存しません。 |
| ✅ | **動的な日付変数** | `{{today}}`、`{{yesterday}}`、`{{month_start}}`、`{{date:-7}}` —— レポートは常に正しい期間を照会します。 |
| ✅ | **信頼性ステップ** | テキスト／要素／ページ安定の待機、フローの自動整理、よくある落とし穴を指摘する静的フロー診断。 |
| ✅ | **スケジュール実行** | 手動、毎日、平日、または N 分ごと。 |
| ✅ | **スクリーンショットと通知** | 成功・失敗、または人手が必要なとき（2FA／CAPTCHA／承認）に通知。 |
| ✅ | **ローカルファースト＆プライベート** | タスク・ログ・設定はブラウザ内に保存。JSON でバックアップ／復元。 |
| ✅ | **サイトごとの権限パネル** | ホストアクセスをサイト単位で確認・許可・取り消し。 |
| ✅ | **任意の AI** | OpenAI／Anthropic／Gemini／DeepSeek／Ollama／Groq に接続可能 —— 既定はオフ、あくまで補助。 |

➡️ 公開済みと計画中の機能（信頼性アダプター、失敗ステップからの再開、AI 修復、チーム同期）は **[ロードマップ](ROADMAP.md)** を参照してください。

## 🆚 WebLoop と重量級 RPA の比較

| | WebLoop | 従来の RPA |
| :--- | :--- | :--- |
| **規模** | Chrome 拡張機能 | デスクトップ／VM へのインストール |
| **学習コスト** | クリックして記録するだけ | トレーニングと認定 |
| **コスト** | 無料・オープンソース（MIT） | 高額なライセンス |
| **プライバシー** | ローカルファースト、サイト単位の許可 | 多くはクラウド／広範な権限 |
| **Web 互換性** | 複雑な企業 DOM 向けに設計 | Web アプリで壊れやすい |
| **AI** | 任意、コアは決定論的 | 後付け |

## 📖 使い方

1. **記録** —— 対象ページを開き、サイドパネルで「記録開始」を押し、作業を一度だけ実行します。
2. **調整** —— 待機を追加し、日付を変数化し、フロー診断に弱点を指摘させます。
3. **テスト** —— 一度（または任意のステップから）実行し、ステップごとのログを確認します。
4. **スケジュールして手放す** —— 「毎日／平日／間隔」を選択。WebLoop がバックグラウンドで実行し、通知します。

詳しい手順は **[docs/USER_GUIDE.md](docs/USER_GUIDE.md)** を参照してください。

## 🛠 インストール

WebLoop のサイドパネルは Vite でビルドする React + TypeScript アプリです。一度ビルドし、生成された `dist/` フォルダーを読み込みます。

```bash
npm install
npm run build      # 型チェック、サイドパネルのビルド、dist/ の組み立て
```

1. `chrome://extensions/` を開く
2. **デベロッパーモード** を有効にする
3. **パッケージ化されていない拡張機能を読み込む** → 生成された **`dist/`** フォルダーを選択

> 決定論的なコア（`service_worker.js`、`content_script.js`）はプレーンな JavaScript で、`dist/` に**そのまま**コピーされます —— バンドルされるのは UI だけです。

### 🧑‍💻 開発

```bash
npm run dev        # Vite 開発サーバー
npm run typecheck  # tsc --noEmit
npm run build      # 本番ビルド → dist/
npm run package    # ビルドしてストア用 zip を作成
```

## 🔐 権限とプライバシー

WebLoop は**ローカルファースト**で、「すべてのサイトのデータを読み取る」といった広範なアクセスを
事前に要求しません。**サイト単位で**、しかもそのサイトで記録・実行するときにのみ許可を求め、
すべての許可はアプリ内の**権限**タブからいつでも取り消せます。各基本権限は
**[docs/PERMISSIONS.md](docs/PERMISSIONS.md)** で一つずつ説明しています。

## 🧱 技術とアーキテクチャ

- **サイドパネル：** React 18 + TypeScript、Vite でバンドル。
- **自動化コア：** 依存の少ないプレーン JS の Chrome **Manifest V3** サービスワーカー＋コンテンツスクリプト。監査しやすいよう `dist/` にそのままコピーされます。
- **ストレージ：** `chrome.storage.local` —— バックエンドもテレメトリーもなし。

詳細は **[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)** · コントリビュートは **[CONTRIBUTING.md](CONTRIBUTING.md)**。

## ❓ FAQ（AI 検索 / GEO 向けも兼ねて）

**WebLoop とは？**
WebLoop は無料・オープンソースの**ノーコード・ブラウザ自動化** Chrome 拡張機能で、繰り返しの
Web ワークフロー（フォーム入力、フィルタリング、クリック、ファイルのダウンロード、スクリーンショット、
通知）を記録・再生します。企業やレガシー Web アプリ向けの**軽量でローカルファーストな RPA** 代替です。

**API や LLM は必要ですか？**
いいえ。コアの記録・再生ループは完全に決定論的で、API も LLM も不要でローカルに動作します。
AI 補助は任意で、あくまで提案のみです。

**データはどこかに送信されますか？**
既定では、何もブラウザの外に出ません。任意の AI アシスタントを明示的に有効にした場合のみ、
**あなたが設定した**プロバイダーへサニタイズ済みの抜粋が送られます。

**UiPath / Power Automate / Automa / Bardeen との違いは？**
WebLoop はあえて小さく焦点を絞っています。ローカルファースト、決定論的、サイト単位の権限、
そして雑然とした企業ページに対する優れた可観測性 —— フル機能の RPA IDE ではありません。

**どのサイトで動きますか？**
あなたが許可した任意の `http(s)` ページ —— ERP/OA/CRM システム、イントラネット、レポートポータル、
その他、使える API のない Web アプリ。
