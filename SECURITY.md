# Security Policy

WebLoop is intended for sensitive enterprise web workflows. Security and privacy should be treated as product requirements.

## Supported versions

Only the latest development version is currently supported.

## Reporting a vulnerability

Open a private security advisory or contact the maintainer directly if repository settings allow it.

## Security rules

- Do not add features that bypass CAPTCHA, 2FA, or anti-fraud controls.
- Do not execute arbitrary remote JavaScript.
- Do not upload page content by default.
- Keep site access permission scoped to user-approved origins.
- Treat screenshots and logs as sensitive artifacts.
