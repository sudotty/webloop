## 😩 Проблема

У большинства внутренних бизнес-приложений — ERP, CRM, OA, интранетов, порталов отчётности —
**нет API**. Поэтому люди сами становятся API: каждое утро кто-то открывает одну и ту же
страницу, выбирает вчерашнюю дату, применяет одни и те же фильтры, нажимает *Запрос*, ждёт
медленную таблицу, нажимает *Экспорт* и отправляет файл по почте. Каждый. Божий. День.

Тяжёлые RPA-системы (UiPath, Power Automate) дороги, требуют обучения и установки на рабочий
стол. Чистые «ИИ-агенты» галлюцинируют и недетерминированы. Облачные рекордеры, которые
требуют *«прочитать все ваши данные»*, отклоняются службой информационной безопасности.

## 💡 Решение

> **WebLoop рассматривает DOM браузера как API.** Если вы можете на это кликнуть — вы можете это автоматизировать.

Лёгкое расширение для Chrome, которое **один раз записывает ваши реальные клики** и
воспроизводит их детерминированно — по запросу или по расписанию — полностью **на вашем компьютере**.

## ✨ Возможности (доступны уже сегодня)

| | Возможность | Что она делает |
| :-- | :--- | :--- |
| ✅ | **Запись без кода** | Фиксирует клики, ввод, выбор, наведения, двойные клики, флажки и загрузки. |
| ✅ | **Детерминированное воспроизведение** | Элементы с несколькими локаторами (CSS, XPath, стабильный текст, aria-label) + оценка уверенности — никаких галлюцинирующих агентов. |
| ✅ | **Динамические переменные даты** | `{{today}}`, `{{yesterday}}`, `{{month_start}}`, `{{date:-7}}` — отчёты всегда запрашивают нужный период. |
| ✅ | **Шаги надёжности** | Ожидание текста / элемента / стабильной страницы; автоматическая очистка процесса; статический аудит процесса, выявляющий типичные ловушки. |
| ✅ | **Планирование** | Вручную, ежедневно, по рабочим дням или каждые N минут. |
| ✅ | **Скриншоты и уведомления** | При успехе, сбое или когда требуется участие человека (2FA / CAPTCHA / согласование). |
| ✅ | **Приоритет локальной работы и приватность** | Задачи, журналы и настройки остаются в вашем браузере. Резервное копирование/восстановление в формате JSON. |
| ✅ | **Панель разрешений для каждого сайта** | Просматривайте, предоставляйте и отзывайте доступ к хостам по одному сайту за раз. |
| ✅ | **Опциональный ИИ** | Подключите OpenAI / Anthropic / Gemini / DeepSeek / Ollama / Groq — по умолчанию отключён, только в роли советчика. |

## 📖 Как это работает

1. **Записать** — откройте нужную страницу, нажмите *Начать запись* на боковой панели и выполните задачу один раз.
2. **Доработать** — добавьте ожидания, превратите даты в переменные и позвольте аудиту процесса выявить слабые места.
3. **Протестировать** — запустите один раз (или с любого шага) и прочитайте журналы по каждому шагу.
4. **Запланировать и расслабиться** — выберите *Ежедневно / Рабочие дни / Интервал*; WebLoop работает в фоне и уведомляет вас.

Полное руководство: **[docs/USER_GUIDE.md](docs/USER_GUIDE.md)**.

## 🛠 Установка

Боковая панель WebLoop — это приложение на React + TypeScript, собранное с помощью Vite. Соберите один раз, затем загрузите сгенерированную папку `dist/`.

```bash
npm install
npm run build      # type-checks, builds the side panel, assembles dist/
```

1. Откройте `chrome://extensions/`
2. Включите **Режим разработчика**
3. **Загрузить распакованное расширение** → выберите сгенерированную папку **`dist/`**

> Детерминированное ядро (`service_worker.js`, `content_script.js`) — это чистый JavaScript, копируемый в `dist/` **дословно** — собирается только интерфейс.

### 🧑‍💻 Разработка

```bash
npm run dev        # Vite dev server
npm run typecheck  # tsc --noEmit
npm run build      # production build → dist/
npm run package    # build + zip a store-ready archive
```

## 🔐 Разрешения и приватность

WebLoop работает с **приоритетом локальной обработки** и никогда не запрашивает широкий доступ
вида *«прочитать все ваши данные»*. Он запрашивает доступ **по одному сайту за раз**, только
когда вы записываете или запускаете процесс на нём, и любое предоставление можно отозвать на
вкладке **Разрешения** внутри приложения. Каждое базовое разрешение построчно объясняется в
**[docs/PERMISSIONS.md](docs/PERMISSIONS.md)**.

## 🧱 Технологии и архитектура

- **Боковая панель:** React 18 + TypeScript, собрано с помощью Vite.
- **Ядро автоматизации:** service worker + content script на чистом JavaScript с минимумом зависимостей для Chrome **Manifest V3**, копируется в `dist/` без изменений для возможности аудита.
- **Хранилище:** `chrome.storage.local` — без бэкенда, без телеметрии.

## ❓ FAQ (в том числе для ИИ-поиска / GEO)

**Что такое WebLoop?**
WebLoop — это бесплатное расширение Chrome с открытым исходным кодом для **автоматизации
браузера без кода**, которое записывает и воспроизводит повторяющиеся веб-процессы —
заполнение форм, фильтрацию, клики, загрузку файлов, скриншоты и уведомления — как
**лёгкая RPA-альтернатива с приоритетом локальной работы** для корпоративных и устаревших
веб-приложений.

**Нужен ли ему API или LLM?**
Нет. Основной цикл записи и воспроизведения полностью детерминирован и работает локально без
какого-либо API или LLM. Помощь ИИ опциональна и носит только рекомендательный характер.

**Отправляются ли мои данные куда-либо?**
По умолчанию ничто не покидает ваш браузер. Только если вы явно включите опциональный
ИИ-ассистент, очищенный фрагмент будет отправлен провайдеру, которого настроите *вы*.

**Чем он отличается от UiPath / Power Automate / Automa / Bardeen?**
WebLoop намеренно компактен и сфокусирован: приоритет локальной работы, детерминированность,
разрешения по сайтам и первоклассная наблюдаемость для запутанных корпоративных страниц — это
не полноценная RPA-IDE.

**На каких сайтах он работает?**
На любой странице `http(s)`, которую вы авторизуете, — системы ERP/OA/CRM, интранеты, порталы
отчётности и другие веб-приложения без пригодного к использованию API.
