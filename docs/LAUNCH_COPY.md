# Launch Copy

## GitHub About description

Record repetitive browser workflows once. Replay forms, filters, clicks, downloads, screenshots, and notifications anytime. LLM-ready, not LLM-dependent.

## Short pitch

WebLoop is a lightweight Chrome extension for repetitive web tasks. It records browser actions and replays them on demand or on schedule. It is built for enterprise web apps, intranet systems, ERP/OA/CRM pages, reporting portals, and other workflows that still require manual clicking and form filling.

## Product Hunt style pitch

Record once. Run anytime.

WebLoop automates repetitive browser workflows: fill forms, select filters, click buttons, export files, capture screenshots, and get notified when the job succeeds or fails. It works locally without LLMs, with optional AI-assisted repair planned.

## Hacker News style pitch

I built a small Chrome extension for deterministic browser workflow recording and replay. The goal is not to replace full RPA suites, but to make daily enterprise web tasks easier: fill the same filters, click query, wait for a slow table, export Excel, and notify on failure. It is plain MV3, no build step, no model dependency.

## Reddit style pitch

If you repeat the same browser workflow every day in an ERP, CRM, OA, intranet, or reporting portal, WebLoop records the steps and replays them later. It supports forms, filters, hover menus, downloads, screenshots, scheduling, and notifications. It does not require an API or LLM.
